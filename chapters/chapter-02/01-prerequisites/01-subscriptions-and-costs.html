Subscriptions and Costs

Coding Plan Providers and the Associated Costs

Powering Claude Code requires more than just the harness — it demands a strong underlying model with the right balance of capability, speed, and quota generosity. This section compares the four leading coding-specific subscription plans (Anthropic, Kimi, Z.ai, and MiniMax) as of mid-April 2026, revealing exactly how their wildly different pricing, 5-hour limits, and metering units (tokens vs. prompts vs. requests) translate into real-world coding productivity.

Whether you want maximum volume on a budget, generous prompt-based headroom, or the absolute best model quality, you’ll find a clear value hierarchy and practical decision framework to pick the plan that fits how you actually code.

Subsection 1/3 ~10 min read Claude Code ≥ 2.1

First things first, Claude Code is only a harness that facilitates the coding workflows, thus it needs a powerful AI model to run it. Without a capable underlying large language model providing strong reasoning, tool-calling, and long-context capabilities, even the most sophisticated agentic features and tool integrations cannot deliver results.

Claude Code is made and maintained by Anthropic, so the default and easiest choice is to go with Anthropics Claude models. The integration is seamless, the feature set is fully optimized, and you get immediate access to the latest Claude family releases, including high-performance variants like Claude Opus and Sonnet.

However, there is a multitude of coding plans that work very good with Claude Code. The harness is flexible enough to support a wide range of providers through standard API configuration. You can connect Claude Code to any of these subscriptions, and it will run just fine, often with only minimal setup changes such as environment variables or a settings file.

You can even run Claude Code with your locally deployed models, but that is a story for another section.

Obviously, some models are better than others and some work better with Claude Code than others. Performance varies significantly depending on the model's reasoning capabilities, tool-calling reliability, context window size, and how well it handles long-running agentic tasks. Almost all of these require you to have a paid subscription.

In this subsection we present the best options for the subscriptions that can power Claude Code.

Price and Quota Comparison

All major providers offering subscriptions compatible with Claude Code use a prepaid subscription model. You pay in advance and receive fixed usage quotas that typically reset on a continuous 5-hour rolling window.

  • Anthropic: Monthly billing for all plans ($20 Pro, $100 Max 5x, $200 Max 20x). Annual billing available only for Pro ($17/month when paid yearly).
  • Z.ai (GLM): Monthly, quarterly and yearly options available. Quarterly and yearly billing provides significantly better value.
  • MiniMax: Monthly billing by default, with annual options offering discounts. Furthermore, a high-speed variant (2x generation speed) is also offered at a significantly higher price (4x/3x).
  • Kimi (Moonshot): Monthly and yearly billing options. Four plans ranging from 20$ to 200$.

Key things to watch out for:

  • Most plans use 5-hour rolling limits rather than fixed daily resets.
  • Usage quotas are usually shared with the provider’s regular web chat interface.
  • There are generally no overage fees, you simply cannot exceed the quota until it refreshes.
  • Subscriptions auto-renew by default.
  • Quarterly and yearly billing often deliver the best price-per-usage.
  • Payment methods: International credit cards are widely supported, but occasional regional issues have been reported with some providers.

Decoding Usage Quotas

When evaluating Claude Code subscriptions, most developers instinctively look at the monthly price first. That part is straightforward, it’s listed in clean USD figures and tells you exactly what you’ll be billed. But the real story, and the one that determines whether a plan actually feels generous or restrictive in daily use, lies in the 5-hour Limit column of the comparison table.

This column is where things get nuanced. Across the four major providers — Anthropic, Z.ai/GLM, MiniMax, and Kimi — the 5-hour limits are expressed in four different units: Messages, Prompts, Requests, and API Calls. At first glance these terms sound interchangeable, especially in casual conversations about AI coding tools. In reality, they represent fundamentally different abstractions that providers use to meter usage. Getting the distinctions right is essential because a single “user action” in Claude Code (typing a task description and hitting enter) can translate into wildly different consumption depending on which backend you’re connected to.

Let’s break it down clearly so you can make an informed decision.

Anthropic: Token-Weighted “Messages”

With Anthropic’s plans, the term “Messages” in the 5-hour limit column does not refer to a fixed number of full conversations or even individual prompts you send through Claude Code. Instead, it represents a token budget. You are allocated a certain number of tokens that can be consumed within any rolling 5-hour window.

The numbers shown in comparison tables are calculated based on very light interactions — the kind of quick, low-token exchanges that barely touch the model’s capacity. In practice, this means real-world usage often feels much tighter than the headline number suggests. Implementing a substantial new feature can easily burn through your quota in just 5–10 messages. Heavy tasks, such as a large-scale database refactor or a complex architectural overhaul, can exhaust the entire 5-hour limit in a single message because of the massive token count involved in reading, reasoning over, and rewriting large codebases.

Z.ai/GLM: Full-Turn “Prompts”

Z.ai (powered by GLM models) uses a very forgiving abstraction: the Prompt. Here, one prompt equals one complete user task submitted through the Claude Code interface, from the moment you hit enter until the agent finishes its final response.

What makes this especially powerful is that all the internal model invocations that happen during that task count as a single prompt. The entire agentic loop — including tool calls, reasoning steps, code editing, testing, re-reading files, and follow-up planning — is bundled together. No matter how many tokens are burned or how many back-and-forth cycles the agent performs, it still registers as only one prompt against your quota.

Z.ai’s own documentation confirms that supported coding tools (Claude Code, Kilo Code, and similar integrations) come with an allowance of tens of billions of tokens per month — effectively around 1% of standard API pricing. In the 5-hour window this typically translates to roughly 80 prompts, giving you enormous headroom for complex, long-running tasks.

MiniMax and Kimi: Requests / API Calls

MiniMax and Kimi (Moonshot) both measure usage in Requests (sometimes labeled API Calls). These are the most granular units: each individual HTTP request sent to the provider counts as one unit.

In Claude Code’s agentic workflow, a single user task typically triggers multiple requests because the system breaks work down into discrete steps: read context → plan → edit files → run tests → re-read changed code → reason about next steps → edit again, and so on. For an average task you might see anywhere from 5 to 20 requests. More complex or iterative work can scale far higher, limited only by the task’s inherent complexity and the model’s context window.

Because each step is metered separately, the effective number of “user-level tasks” you can complete in a 5-hour window is lower than with prompt-based systems, but the per-request limits themselves are often quite high (MiniMax, for example, offers 4,500 requests every 5 hours with no weekly cap).

Which Subscription Offers the Best Value for Money?

After mapping the units back to practical developer workflows, a clear value hierarchy emerges, at least when measured in value delivered for money spent.

MiniMax Plus ($20/mo) currently delivers the single best raw usage-per-dollar for most Claude Code users. Its 4,500 requests every 5 hours (with no weekly cap) dwarfs what you get from Anthropic Pro or even the Max 5× tier at the price point of a basic package.

Z.ai Lite/Pro comes in a very close second and, especially when paid quarterly or yearly, often feels like 3–5× the effective usage of Claude Pro/Max at a significantly lower cost. Many developers keep Z.ai as their “Claude Code overflow” plan, switching seamlessly the moment Anthropic’s limits are reached. The prompt-based metering makes it feel exceptionally generous for long, complex coding sessions.

Anthropic still wins on raw quality and the seamless, polished experience that comes with native integration and the absolute best reasoning capabilities (especially when using Claude Opus). However, it loses badly on pure usage value. The Max 20× plan is only worth considering if you specifically need the highest-end model and have the budget ($200/mo) to support the much tighter token-based limits.

Kimi (Moonshot) sits comfortably in the middle of the pack. It offers solid weekly quotas and excellent compatibility with Claude Code, but its reset structure is less forgiving for bursty, all-day coding sessions compared to the continuous 5-hour rolling windows offered by the other three providers.

Final Takeaway for Decision-Making

The best subscription ultimately depends on your specific workflow:

  • If you want maximum volume and don’t mind managing the agentic-loop granularity, go with MiniMax Plus.
  • If you want maximum simplicity and generous prompt-based headroom at a very attractive price, Z.ai Pro is the sweet spot for most heavy users.
  • If quality and frictionless experience are non-negotiable and budget is secondary, Anthropic Max (with Opus) remains the premium choice.
  • Kimi serves as a reliable middle-ground option when you need balanced quotas without extreme volume or extreme cost.
